# 📊 샘플 Google Sheets 데이터

> 이 문서는 각 매장 Google Sheets에 실제로 입력될 데이터의 형식과 예시를 보여줍니다.
> 13개 컬럼(필수 7 + 선택 6) 구조입니다.

---

## 컬럼 구조

| # | 컬럼명 | 필수 | 타입 | 설명 |
|---|--------|------|------|------|
| 1 | 주문일 | ✅ | DATE | `YYYY-MM-DD` 형식 |
| 2 | 매장코드 | ✅ | STRING | settings.yaml의 stores[].code와 일치 |
| 3 | 매장명 | ✅ | STRING | 한글 매장명 |
| 4 | 원두코드 | ✅ | STRING | settings.yaml의 beans[].code와 일치 |
| 5 | 원두명 | ✅ | STRING | 한글 원두명 |
| 6 | 수량(kg) | ✅ | NUMBER | 양의 숫자, 소수점 첫째 자리까지 허용 |
| 7 | 배송희망일 | ✅ | DATE | `YYYY-MM-DD` 형식 |
| 8 | 로스팅단계 | ⬜ | STRING | `라이트` / `미디엄` / `다크` |
| 9 | 분쇄여부 | ⬜ | STRING | `홀빈` / `분쇄` |
| 10 | 긴급여부 | ⬜ | STRING | `Y` / `N` |
| 11 | 비고 | ⬜ | STRING | 자유 입력 |
| 12 | 주문자 | ⬜ | STRING | ❌ 리포트 미출력 |
| 13 | 연락처 | ⬜ | STRING | ❌ 리포트 미출력 |

---

## 서면본점 Sheets 예시

| 주문일 | 매장코드 | 매장명 | 원두코드 | 원두명 | 수량(kg) | 배송희망일 | 로스팅단계 | 분쇄여부 | 긴급여부 | 비고 | 주문자 | 연락처 |
|--------|---------|--------|---------|--------|---------|-----------|-----------|---------|---------|------|--------|--------|
| 2026-02-15 | SM | 서면본점 | ETH-YRG | 에티오피아 예가체프 | 5 | 2026-02-17 | 미디엄 | 홀빈 | N | | 김철수 | 010-1234-5678 |
| 2026-02-15 | SM | 서면본점 | COL-SUP | 콜롬비아 수프리모 | 3 | 2026-02-16 | 다크 | 분쇄 | Y | 급한 주문 | 이영희 | 010-9876-5432 |

## 양산 Sheets 예시

| 주문일 | 매장코드 | 매장명 | 원두코드 | 원두명 | 수량(kg) | 배송희망일 | 로스팅단계 | 분쇄여부 | 긴급여부 | 비고 | 주문자 | 연락처 |
|--------|---------|--------|---------|--------|---------|-----------|-----------|---------|---------|------|--------|--------|

> ⚠️ **참고**: 주문자·연락처 컬럼은 원본 Sheets에만 존재하며, 리포트에는 **절대 출력하지 않습니다** (개인정보 보호 원칙).
